# Environment Variables

This app uses Vite env variables. Client-visible values must start with `VITE_`.

## Required SDK variables

Application-side SDK integration uses dual-token mode:
- `Authorization: Bearer <authToken>` (user auth token, runtime)
- `Access-Token: <accessToken>` (app access token or runtime access token)

Configured env variables:

- `VITE_API_BASE_URL`
  - Production default: `https://api.sdkwork.com`
  - Non-production default: `https://api-dev.sdkwork.com`
- `VITE_ACCESS_TOKEN`
  - SDKWork app development Access Token

## Compatibility variables

To keep existing modules working, these aliases are still supported:
- `VITE_APP_API_BASE_URL` (fallback of `VITE_API_BASE_URL`)
- `VITE_APP_IM_WS_URL` (fallback of `VITE_IM_WS_URL`)

## Mode files

- `.env.production`
- `.env.development`
- `.env.test`
- `.env.dev` (legacy alias mode)
- `.env` (base defaults)
