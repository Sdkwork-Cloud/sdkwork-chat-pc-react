# Environment Variables

This app uses Vite env variables. Client-visible values must start with `VITE_`.

## Required SDK variables

Application-side SDK integration uses strict token separation:
- `authToken`: user login/session token (runtime only, never configured in env)
- `accessToken`: app access token (configured in env/bootstrap)

Configured env variables (standardized across apps):

- `VITE_APP_ENV`
- `VITE_API_BASE_URL`
- `VITE_TIMEOUT`
- `VITE_API_KEY`
- `VITE_ACCESS_TOKEN`
- `VITE_TENANT_ID`
- `VITE_ORGANIZATION_ID`
- `VITE_PLATFORM`

## Compatibility variables

To keep existing modules working, these aliases are still supported:
- `VITE_APP_API_BASE_URL` (fallback of `VITE_API_BASE_URL`)
- `VITE_APP_IM_WS_URL` (fallback of `VITE_IM_WS_URL`)
- `SDKWORK_API_BASE_URL`, `SDKWORK_ACCESS_TOKEN`

## Mode files

- `.env.example`
- `.env.production`
- `.env.development`
- `.env.test`
- `.env.dev` (legacy alias mode)
- `.env` (base defaults)
