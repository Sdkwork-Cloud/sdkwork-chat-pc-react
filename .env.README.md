# Environment Variables

This app uses Vite env variables. Client-visible values must start with `VITE_`.

## Required SDK variables

Application-side SDK integration uses strict token separation:
- `authToken`: user login/session token (runtime only, never configured in env)
- `accessToken`: app access token (configured in env/bootstrap)

Configured env variables (standardized across apps):

- `VITE_APP_ENV`
- `VITE_API_BASE_URL`
- `VITE_TIMEOUT`
- `VITE_API_KEY`
- `VITE_ACCESS_TOKEN`
- `VITE_TENANT_ID`
- `VITE_ORGANIZATION_ID`
- `VITE_PLATFORM`

## Compatibility variables

To keep existing modules working, these aliases are still supported:
- `VITE_APP_API_BASE_URL` (fallback of `VITE_API_BASE_URL`)
- `VITE_APP_IM_WS_URL` (fallback of `VITE_IM_WS_URL`)
- `SDKWORK_API_BASE_URL`, `SDKWORK_ACCESS_TOKEN`

## Mode files

- `.env.example`
- `.env.production`
- `.env.development`
- `.env.test`
- `.env.dev` (legacy alias mode)
- `.env` (base defaults)

## Desktop Release Notes

- Tauri desktop builds now read the Vite mode from `SDKWORK_VITE_MODE`.
- `pnpm run dev:desktop`, `pnpm run dev:desktop:test`, `pnpm run build:desktop:dev`, `pnpm run build:desktop:test`, and `pnpm run build:desktop:prod` select the corresponding env file automatically.
- Shared SDK sourcing is mode-based:
  - local development defaults to `SDKWORK_SHARED_SDK_MODE=source` and resolves sibling source paths
  - CI and release use `SDKWORK_SHARED_SDK_MODE=git` and materialize the SDK from GitHub before install/build
  - release automation should pin `SDKWORK_SHARED_SDK_APP_GIT_REF` and `SDKWORK_SHARED_SDK_COMMON_GIT_REF` to immutable refs so `pnpm install --frozen-lockfile` remains reproducible
